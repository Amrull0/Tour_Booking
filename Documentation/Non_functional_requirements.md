### Нефункциональные требования

#### Простота использования
- Интерфейс должен быть понятен без предварительного обучения
- Все основные действия доступны в 2–3 клика  

#### Безопасность
- Доступ к функциям приложения предоставляется только авторизованным пользователям.
- Защита регистрации кодом доступа  
- Разграничение прав пользователей и администраторов

#### Производительность
- Приложение должно корректно сохранять данные пользователей и бронирования в облачной базе Firebase Firestore
- Список туров и данные пользователя должны быстро загружатся
- Быстрый поиск по базе данных
