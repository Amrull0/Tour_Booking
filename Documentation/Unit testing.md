## Тест-кейсы для модульного тестирования 

### Таблица 1: Тестирование валидации email при регистрации
| ID | Цель | Входные данные | Ожидаемый результат |
|:---|:---|:---|:---|
| 1 | Проверка корректного email | Email: "user@example.com" | Метод возвращает true (Email валиден) |
| 2 | Проверка email без @ | Email: "userexample.com" | Метод возвращает false (Отсутствует @) |
| 3 | Проверка email без домена | Email: "user@" | Метод возвращает false (Нет доменной части) |
| 4 | Проверка пустого email | Email: "" | Метод возвращает false (Email не может быть пустым) |

### Таблица 2: Тестирование валидации цены тура
| ID | Цель | Входные данные | Ожидаемый результат |
|:---|:---|:---|:---|
| 1 | Проверка корректной цены | Цена: 50000.0 | Метод возвращает true (Цена валидна) |
| 2 | Проверка отрицательной цены | Цена: -1000.0 | Метод возвращает false (Цена должна быть положительной) |
| 3 | Проверка нулевой цены | Цена: 0.0 | Метод возвращает false (Цена должна быть больше 0) |
| 4 | Проверка слишком большой цены | Цена: 10000000.0 | Метод возвращает true (Нет ограничения сверху) |

### Таблица 3: Тестирование валидации количества участников бронирования
| ID | Цель | Входные данные | Ожидаемый результат |
|:---|:---|:---|:---|
| 1 | Проверка корректного количества | Участники: 2 | Метод возвращает true (Количество валидно) |
| 2 | Проверка минимального количества | Участники: 1 | Метод возвращает true (Минимум 1 участник) |
| 3 | Проверка нулевого количества | Участники: 0 | Метод возвращает false (Не может быть 0 участников) |
| 4 | Проверка максимального количества | Участники: 50 | Метод возвращает false (Максимум 30 участников) |
