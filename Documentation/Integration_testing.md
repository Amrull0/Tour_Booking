# Тест-кейсы для интеграционного тестирования

| ID | Цель | Действие | Ожидаемый результат |
|:---|:---|:---|:---|
| **1** | Полный цикл бронирования тура | Админ создает новый тур, пользователь бронирует его, админ подтверждает бронирование, пользователь видит статус "Подтверждено". | Тур появляется в общем списке, бронирование создается, статус последовательно меняется с "Ожидание" на "Подтверждено". |
| **2** | Добавление в избранное и синхронизация | Пользователь добавляет тур в избранное на одном устройстве, входит с другого устройства под тем же аккаунтом. | Избранный тур отображается на обоих устройствах, иконка заполненной звезды сохраняется. |
| **3** | Разграничение прав пользователя и админа | Обычный пользователь пытается открыть вкладку "Управление турами" и "Подтверждение бронирований". | Админские вкладки скрыты из меню навигации, доступ к ним невозможен через прямой переход. |
| **4** | Валидация данных при создании тура | Администратор заполняет форму добавления тура с пустым названием и отрицательной ценой, нажимает "Сохранить". | Появляются сообщения об ошибках, тур не сохраняется в Firebase, форма остается открытой. |
| **5** | Изменение статуса бронирования | Админ в разделе "Подтверждение" меняет статус бронирования с "Ожидание" на "Отменено". | Пользователь в своем разделе "Брони" видит обновленный статус "Отменено" без перезагрузки приложения. |
| **6** | Работа с изображениями туров | Администратор добавляет тур с корректной ссылкой на изображение, пользователь просматривает этот тур в списке и деталях. | Изображение успешно загружается через Glide в обоих местах, отображается placeholder при ошибке загрузки. |
| **7** | Обработка офлайн-режима | Пользователь создает бронирование при отсутствии интернета, затем подключается к сети. | Бронирование локально сохраняется, после подключения синхронизируется с Firebase, появляется в общем списке. |
