# 3. Техническое задание

## 3.1. Цель разработки

Целью разработки является создание **мобильного приложения для бронирования туристических туров**, которое обеспечивает удобный и безопасный доступ к туристическим предложениям. Приложение должно позволять пользователю быстро искать и бронировать туры, а администратору — управлять турами и контролировать бронирования.  

Приложение разрабатывается с использованием современных технологий и обеспечивает:

* Разграничение ролей и функций для пользователя и администратора.
* Сохранение и синхронизацию данных через облачную базу данных.
* Интуитивно понятный и адаптивный интерфейс для мобильных устройств.

---

## 3.2. Описание системы

Приложение представляет собой решение на базе Android для бронирования туров. Система объединяет в себе функции клиентской платформы и административного управления турами.

Основные модули системы:

Клиентский модуль: просмотр списка туров с фильтрацией по стране, дате и стоимости, просмотр подробной информации о турах, оформление и отмена бронирований, отслеживание статуса заказов и история прошлых бронирований.
Модуль управления турами: добавление новых туров, редактирование существующих и контроль актуальности информации о турах.
Модуль администрирования: управление бронированиями клиентов, подтверждение или изменение заказов, просмотр статистики по бронированиям, доступ ко всем функциям клиента для контроля и поддержки работы системы.

---

## 3.3. Целевая аудитория

Целевая аудитория приложения включает:

* **Пользователей (туристов):** люди, которые хотят быстро и удобно искать туры, оформлять бронирования и отслеживать статус своих поездок.  
* **Администраторов (сотрудников туристического агентства):** лица, управляющие предложениями туров, контролирующие бронирования и обеспечивающие актуальность данных.

Приложение ориентировано на широкую аудиторию пользователей мобильных устройств, включая людей с разными уровнями технической подготовки.

---

## 3.4. Общие требования
**Обеспечение стабильной работы и безопасности данных:**  
  Для корректной работы приложения и защиты информации предъявляются следующие требования:

  * **Технологический стек:** приложение разрабатывается на языке **Java** для платформы **Android** с использованием облачной базы данных **Firebase Firestore** для хранения и синхронизации данных.

  * **Доступ и авторизация:** функции приложения доступны только авторизованным пользователям. Регистрация новых аккаунтов осуществляется с использованием мастер-пароля, чтобы предотвратить несанкционированный доступ.

  * **Интерфейс и удобство использования:** графическая оболочка должна быть интуитивно понятной, обеспечивать быстрый доступ к спискам туров и элементам бронирования, а также соответствовать современным стандартам мобильной разработки.
